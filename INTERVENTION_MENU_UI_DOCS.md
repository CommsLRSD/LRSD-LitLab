## Key Design Principles

### Minimalism
- Clean white backgrounds with subtle shadows
- Clear visual hierarchy with typography
- Minimal use of colors (primary blue accent)
- No clutter - one clear action per screen

### Progressive Disclosure
- Users only see relevant options at each step
- Subtests filtered by selected screener
- Results filtered by all previous selections
- No overwhelming choices

### Clear Navigation
- Back buttons on every step (except step 1)
- Progress indicator shows current position
- Visual feedback on hover/click
- Breadcrumb trail on results page

### Responsive Design
- Card-based layout adapts to screen size
- Mobile-first approach
- Touch-friendly tap targets (48px minimum)
- Readable fonts on all devices

4. **Sorting**: Evidence level (**,*,none) then alphabetical

## Accessibility Features
- Semantic HTML (h3, button, etc.)
- ARIA labels on interactive elements
- Keyboard navigation support
- High contrast ratios
- Focus indicators on all interactive elements

## Browser Compatibility
- Modern browsers (Chrome, Firefox, Safari, Edge)
- CSS Grid and Flexbox for layouts
- ES6+ JavaScript features
- No external dependencies required
